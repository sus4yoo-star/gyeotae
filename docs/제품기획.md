# 곁에 (GYEOTAE) — 제품 비전 & 로드맵 기획서

> **v0.1 · 2026.06**
> 타깃: 일반 가족 (평상시 안부 + 만약의 대비)

---

## 1. 한 줄 비전

> **"평소엔 곁을 잇고, 만약엔 곁을 지킨다."**
> 멀리 사는 가족이 부모님의 평범한 하루를 함께하고, 위급한 순간엔 가장 먼저 닿는 가족 돌봄 앱.

**미션:** 떨어져 사는 가족 사이의 *물리적 거리*를 *마음의 거리*로 바꾸지 않게 한다.

---

## 2. 문제 정의

- 자녀는 멀리서 일하고, 부모님은 혼자 계신다. 매일 전화하긴 부담스럽고, 안 하면 불안하다.
- "별일 없겠지"와 "혹시 무슨 일 있으면?" 사이의 **막연한 불안**이 일상에 깔려 있다.
- 정작 응급 상황이 오면 — 복용약, 지병, 혈액형, 주치의 연락처를 **아무도 즉시 모른다.**
- 카톡은 소통은 되지만 *안부 확인*도 *응급 대비*도 구조적으로 안 된다.

**핵심 인사이트:** 일반 가족이 원하는 건 24시간 감시가 아니라 **"오늘도 평안하시구나"라는 매일의 안심**과 **"만약의 순간 준비돼 있다"는 보험 같은 든든함**이다.

## 4. 포지셔닝

```
        평상시 소통 ← → 만약의 대비
카톡          ●
캘린더/리마인더    ●
시니어 모니터링(B2B)              ●  (무겁고 감시적)
─────────────────────────────────
곁에        ●━━━━━━━━━━━━━●  ← 이 축 전체를 하나로
```

**차별점:** 소통앱은 응급을 못 하고, 응급/모니터링 솔루션은 일상이 차갑다. 곁에는 **"따뜻한 일상"과 "차가운 만약"을 한 앱에서 자연스럽게.**

---

## 5. 제품 원칙

1. **부모님은 3탭 안에** — 시니어 UX 최우선(큰 글씨, 음성, SOS는 항상 한 번에)
2. **안심은 자동으로, 부담은 0** — 매일 알아서 안부 신호(복약·활동), 자녀는 "괜찮음"만 확인
3. **감시 아님, 동행** — 위치·활동은 *공유*지 *추적*이 아니다. 톤이 생명
4. **만약은 평소에 준비된다** — 응급 정보는 위급할 때가 아니라 평소에 채워둔다
5. **공유는 기본이 아니라 선택** — 무엇을 누구에게 보일지는 항상 보내는 사람이 정한다 *(→ §9 공유 범위 모델)*

## 6. 3대 기둥 × (평상시 ↔ 만약)

| 기둥 | 평상시 (매일 여는 이유) | 만약 (설치·결제 이유) |
|---|---|---|
| 🚨 **안전** | 복약·식사 체크, 무응답 감지, 활동 신호 | SOS 단계 알림, 응급 의료카드, 위치공유, 119 연동 |
| 💬 **소통** | AI 안부, 음성메시지, 손주 영상, 가족 피드 | 위급 시 영상통화 즉시연결, 가족 동시 호출 |
| 📖 **추억** | 엄마의 이야기(음성), 가족 앨범, 기념일 | (정서적 lock-in — 떠날 수 없게 하는 자산) |

## 7. 데일리 루프 (리텐션 설계)

```
부모님: 아침 약 체크 → 안부 한마디(음성)
   ↓ (실시간)
자녀: "엄마 약 드셨네 ✓ 온기 +5" → 손주 영상 보냄
   ↓
부모님: 손주 영상 봄 → 오늘의 이야기 1개 녹음
   ↓
자녀: 이야기 쌓임 → 주말에 자서전/영상으로
```

응급은 *드물게* 작동하지만, 이 루프가 **매일의 DAU**를 만든다.

## 8. 로드맵

### Phase 0 — 현재 (완료)

가족 서클·초대, SOS 전체알림, 의료카드(기본), 복약/식사 동기화, 실시간 피드, AI 안부, 온기 지수, parent-device 모드, PWA·푸시.

### Phase 1 — MVP "안심의 완성" (4~6주)

> 목표: *응급 대비*를 진짜로 만들고, *매일 안심* 루프를 닫는다

- **응급 의료카드 심화**: 지병·알러지·복용약·혈액형·주치의·병원·보험·응급연락망 / **잠금화면·QR 카드**(구급대원이 폰에서 즉시 확인)
- **SOS 에스컬레이션**: 가족 알림 → 무응답 N분 → 자동 119 안내 + 위치공유 + "내가 갈게" 응답
- **무응답 감지 고도화**: 일정 시간 활동 없으면 자녀에게 부드러운 알림
- **음성 메시지**: 시니어가 버튼 하나로 녹음 → 가족에게

### Phase 2 — "소통의 깊이, 그러나 각자의 거리" (6~10주)

> **핵심 원칙:** 공유는 기본이 아니라 **선택**이다. 범위는 항상 보내는 사람이 정한다.
> 모든 채팅·앨범·영상·리포트는 아래 [공유 범위 모델](#9-공유-범위audience-모델--phase-2의-척추) 위에 올라간다.

**① 가족 채팅 — "채널은 나뉘고, 부모님 화면은 단순하게"**
- 채널 구조: `온 가족` · `자녀끼리(백스테이지)` · `1:1`
  - 백스테이지 = **부모님께 안 보이는** 보호자 전용 공간 (돌봄 조율·깜짝 준비)
- 시니어용 부모님 채팅: 채널 복잡함 숨김 → 부모님은 **"가족"과 "안부" 두 개만**. 큰 글씨·음성 우선, 답장은 **녹음 버튼 하나**
- 보낸 메시지 **회수/수정**, 읽음 표시는 *부드럽게*(압박 안 되게)

**② 사진·영상 앨범 — "선택 공개"가 기본**
- 앨범별 공개 범위 + **사진 한 장씩 개별 범위** 덮어쓰기
- 올릴 때: *"이 사진 누구에게?"* → 온가족 / 부모님께만 / 자녀끼리 / 나만 보기
- 콘텐츠 권한: 저장·다운로드 허용/금지, 전달 금지, (가능하면) 캡처 알림
- 회수: 올린 사람이 내리면 모두에게서 사라짐
- **손주 콘텐츠 게이트키퍼**: 아이 사진은 **그 아이의 부모가 공개 범위 통제**(아동 프라이버시)
- **나만의 보관함**: 가족과 공유 안 하는 개인 공간(공유 전 정리/초안)

**③ 영상통화 즉시연결**
- 1:1 또는 그룹콜(누구를 부를지 선택), 부모님은 **얼굴 큰 버튼**으로 한 번에
- 위급 시(SOS 연계) **가족 동시 호출**

**④ 활동/건강 리포트 — "부모님 동의가 먼저"**
- 부모님이 직접 공유 범위 설정: `주보호자만` / `온 가족` / `끄기`
- 항목별 토글(복약·식사·활동·수면) — *위치는 기본 OFF*
- 톤: "감시 수치"가 아니라 **"이번 주도 평안하셨어요" 안심 요약**

### Phase 3 — "추억의 자산화" + 확장 (10주~)

- 음성 자서전 아카이브 → 영상(셀라 연동)
- 기념일·제사·생신 알림
- 웨어러블 연동(애플워치 낙상·심박), 활력징후
- 다(多) 부모님 / 대가족

## 9. 공유 범위(Audience) 모델 — Phase 2의 척추

가족앱의 가장 흔한 실패는 *"다 공유"* → 부담스러워 안 쓰게 되는 것. 형제끼리 부모님 걱정 상의, 생신 깜짝 준비, 며느리가 올리기 꺼려지는 사진… **"누구에게 보일지를 보내는 사람이 정한다"**가 핵심.

| 공유 범위 | 누가 보나 | 쓰임새 |
|---|---|---|
| 👨‍👩‍👧 **온 가족** | 서클 전원(부모님 포함) | 일상 안부, 함께 보는 손주 영상 |
| 👵 **부모님께만** | 부모님 + 보낸 사람 | 할머니께 보내는 손녀 영상 |
| 🧑‍🤝‍🧑 **자녀끼리(백스테이지)** | 보호자들만 · **부모님 제외** | 건강 걱정 상의, 생신 깜짝 준비, 병원비 분담 |
| 🔒 **1:1** | 두 사람만 | 형제 간 사담 |
| 📦 **나만 보기(보관함)** | 본인만 | 정리용·공유 전 초안, 감추고 싶은 것 |

- 콘텐츠마다 **올릴 때 범위 선택 + 올린 뒤 회수·범위변경**
- 기본값은 "온 가족"이 아니라 **직전에 고른 범위**(실수 공유 방지)

### 윤리 가드레일 (안 지키면 신뢰 붕괴)

- **"비밀"이 아니라 "배려"**: 자녀끼리 채널은 *뒷담화*가 아니라 *돌봄 조율*용 — 톤·기본 안내로 선 긋기
- **투명성 옵션**: 부모님은 "보호자 전용 공간이 있다"는 *존재 자체*는 알 수 있게(내용은 비공개)
- **삭제권은 올린 사람에게**, 건강정보 통제권은 **부모님에게**
- **위치·건강은 추적이 아니라 공유** — 항상 본인 동의·언제든 끄기

### 데이터 모델 영향 (Phase 2 진입 시)

```
channels      (circle_id, type: family|caregivers|dm, member_ids[])
messages      (channel_id, author, body|voice|media, recalled_at)
media_items   (circle_id, uploader, url, kind,
               visibility: family|parent_only|caregivers|private|custom,
               allow_save bool, audience_ids[], taken_down_at)
albums        (circle_id, visibility, ...)
health_report_consent (circle_id, parent_user_id, scope, fields[])
```

→ 모든 콘텐츠 테이블에 **`visibility` + `audience`**가 1급 시민. **RLS로 *볼 수 있는 사람만* 읽게 강제.**

## 10. 성공 지표

- **North Star:** *주간 안심 액션이 일어난 가족 수* (부모님 신호 1+ & 자녀 확인 1+)
- 보조 KPI: 서클당 활성 가족 수, 복약체크율, D30 리텐션, 의료카드 작성 완료율, SOS 응답시간

## 12. 리스크 & 대응

| 리스크 | 대응 |
|---|---|
| 감시 느낌 → 부모님 거부감 | "동행" 톤, 공유 동의, 부모님 화면 극단순 |
