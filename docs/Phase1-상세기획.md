# Phase 1 상세 기획 — "안심의 완성"

> **기간 4~6주 · 목표:** 응급 대비를 진짜로 만들고, 매일 안심 루프를 닫는다.
> 상위 문서: [`제품기획.md`](./제품기획.md)

---

## 0. 완료 기준 (Definition of Done)

- [ ] 부모님 의료정보가 **DB 저장 → 온 가족 공유**, 구급대원이 **로그인 없이 QR/링크로 즉시 확인**
- [ ] SOS가 단순 알림이 아니라 **상태 있는 사건**(누가 확인했나, 어디인가, 해결됐나) + **에스컬레이션**
- [ ] 부모님이 **일정 시간 무활동이면 자녀에게 부드러운 알림**
- [ ] 부모님이 **버튼 하나로 음성 메시지** 녹음·전송

## 1. 확정된 의사결정 (이 Phase의 전제)

| 항목 | 결정 |
|---|---|
| **119 연동** | 자동신고 ❌ → **원터치 `tel:119` 발신 + 위치 문자 템플릿 보조** |
| **응급뷰 공개 범위** | **응급치료에 필요한 만큼 충분히**: 이름·나이·성별·혈액형·알레르기·지병·현재 복용약·DNR·주치의/병원·보호자 연락처. (보험번호·상세 병력은 가족 전용) |

## 2. 🩺 응급 의료카드 (심화) — 중심 기능

### 유저 스토리
- *자녀로서* 부모님 의료정보를 **대신 채우고** 가족 모두가 본다.
- *구급대원으로서* 의식 없는 환자의 폰/링크에서 **로그인 없이** 핵심 의료정보 + 보호자 전화를 즉시 본다.

### 항목 (9개 → 확장)
신원(이름·나이·성별), **혈액형**, 지병, **알레르기(강조)**, 복용약, 수술이력, 병력, 키/몸무게, **주치의·병원**, **보험**, **응급연락망(다중)**, **DNR(사전연명의료)**, 비고

### 화면 흐름
```
[자녀] 의료카드 → 편집 → DB 저장 → 가족 동기화 ✓
        → "응급 공개 링크/QR" 토글 ON
[부모님 폰] 배경/위젯 = QR + "응급 의료정보"
[구급대원] QR 스캔 → /e/{token} (로그인X) → 핵심정보 + 보호자 [전화걸기]
```

### 데이터 모델 — `supabase/phase1.sql` 참조
`medical_profiles`(circle당 1행, 가족 RLS) + `emergency_links`(공개 토큰) + `get_emergency_card(token)` security-definer RPC(제한 필드만).

### 보안
- `medical_profiles`는 가족만. 익명 응급뷰는 **토큰 RPC로 필요한 필드만**. 토큰 언제든 비활성화.

### 완료 기준
가족 편집→실시간 동기화 / QR 스캔 시 로그인 없이 핵심정보 + 보호자 원터치 전화.

## 3. 🚨 SOS 에스컬레이션

### 상태 머신
```
SOS → sos_alerts(active, 위치) + 가족 전원 푸시
 → 1명 "내가 갈게" → acknowledged, 모두에게 전파
 → N분 무응답 → 재푸시 + tel:119 안내 + 위치문자 템플릿
 → "해결됨" → resolved (피드 기록)
```

### 데이터 모델
`sos_alerts`(status: active|acknowledged|resolved, lat/lng, ack_by, resolved_at) + realtime publication.

### 119 (확정)
**원터치 `tel:119` 발신 + 위치 문자 자동 작성.** 자동신고 아님 — "즉시 발신 + 위치 전달 보조".

### 완료 기준
실시간 상태 전파 / 누가 확인했는지 / 무응답 시 단계상승 / tel:119 + 위치 보조.

---

## 4. 🔕 무응답 감지 고도화

### 정의
부모님 **마지막 활동**(앱 열기·약 체크·안부) 이후 **임계(기본 12h, 야간 제외)** 무활동 → 자녀에 *부드러운* 알림.

### 구현
- `care_circles`에 `last_activity_at` / `silence_threshold_hours` / `quiet_start` / `quiet_end` 추가
- 부모측 액션마다 `last_activity_at = now()`
- **Supabase Cron**: 30분마다 `/api/cron/tick` 호출 → 임계 초과 & 조용시간 아님 & 오늘 미알림 → `silence` 이벤트 + 자녀 푸시

### 완료 기준
임계 초과 시 하루 1회 알림 / 임계값·조용시간 설정 / 활동 발생 시 리셋.

## 5. 🎙️ 음성 메시지

### 구현
- Supabase Storage 비공개 버킷 `voice`
- 부모님: **큰 마이크 버튼** 녹음 → 업로드 → 피드 / 자녀: 재생
- `voice_messages`(가족 RLS, signed URL). **무료 60초·30일 보관**
- 공개 범위는 Phase 1엔 "온 가족" 고정 — Phase 2 audience 모델 진입점만 열어둠

### 완료 기준
버튼 하나로 녹음·전송 / 피드 재생 / 푸시 알림.

---

## 6. 공통 토대 (이번에 도입)
- **Supabase Storage**(음성·추후 사진)
- **스케줄러**(pg_cron + pg_net → `/api/cron/tick`, `CRON_SECRET` 보호)
- **푸시 라우팅 일반화**(SOS 외 알림 타입)

## 7. 빌드 순서 (4~6주) — *전체 동시 진행*
1. 의료카드 DB화 + 가족공유 + 항목심화
2. 응급 공개뷰(`/e/{token}`) + QR
3. SOS 상태머신 + 위치 + ack + 에스컬레이션
4. Storage + 음성 메시지
5. 무응답 감지(cron) + 설정 + 다듬기

## 8. 이번엔 안 함 (Phase 2+)
가족 채팅/채널, 사진 앨범, 영상통화, 공유 범위(audience), 웨어러블, 자서전.

## 9. 운영자가 한 번 해줘야 하는 설정 (구현 후 안내)
1. `supabase/phase1.sql` 을 Supabase SQL Editor에 붙여넣고 RUN (테이블·RLS·RPC·Storage·cron)
